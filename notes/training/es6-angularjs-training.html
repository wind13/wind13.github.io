ES6 AngularJS Training

Schedule(1.5 hour/day)

  • Day 1: Introduce each other, intro modern front-end technology etc. Best practice of GitHub, Agile and review code etc.
  • Day 2: Setup and prepare environment and run demo. Try unit test, E2E test etc, create a new page and a component.
  • Day 3: Create more complex and beautiful pages, connect to back-end RESTful API etc.
  • Day 4: Combine all together and write documents.
  • Day 5 (Optional): Summary, Q & A, Free talk etc.

Lession 1

  • Introduce each other.
  • Talk about develop concept
  • Build your team and project.
  • Intro Git and GitHub
  • Prepare your develop environment
  • Setup Goal! and split tasks and the first start step.
  • List project questions.

Note:

  • 相互介绍，了解对 AngularJS 及前端开发的情况。
  • 此次学习大家将学到很多“课外知识”，甚至很多人生智慧（鸡汤）。鸡汤还是要喝的，万一喝到鸡肉呢！
  • 有输出才能有真正的学习效果。本次将以一个作品做为最终的目标。
  • 设定目标、组建小组。全流程敏捷开发实践。结对编程、TDD 等。
  • 组内设置 Scrum master 和 Product owner 两职位，全部 Developer 兼任。
  • 第一个决定选择 Git 服务。在服务上创建项目，开库，创建任务，规范工作流程。
  • 安装 GitBash 及 VSCode 两个工具。
  • 介绍 markdown 语法和文档书写技巧。
  • 课后作业：
    • 浏览 ES6 语法，重点理解 import, export 与 exports 的区别，整理成文档放在 Git 服务中。
    • 浏览 AngularJS 语法，重点理解 components 一章，整理成文档放在 Git 服务中。

Lession 2

  • Intro Node.js and AngularJS
  • Agile and Scrum practice.
  • Define API document.
  • Write stories.
  • Intro TDD and UDD.
  • Write test case, Unit test and E2E test.
  • Read JavaScript and ES6 document.

Note:

  • 开始了解 AngularJS

Lession 3

  • Start with a skeleton.
  • Introduce the folder and files.
  • Introduce VSCode
  • Introduce AngularJS module, controller, service
  • Introduce Bootstrap 4
  • Implement login component and page
  • Npm install, start and install python server

Lession 4

  • Introduce XXS
  • Introduce vi
  • Introduce design priciple
  • Implement user service
  • Introduce Emmet

Lession 5

  • Introduce filter
  • Thinking in component
  • Try write a filter
  • Use Vali Admin in pages

Home work and exam

Finish user login and user list page.

  • Should use the skeleton;
  • Should use users mock data;
  • Should use a admin mock data to valid login.
  • Normal user should not see edit button.
  • Should use Vali Admin styles.

  • 登录失败并无用户提示，仅有控制台的输出；
