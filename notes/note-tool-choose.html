笔记到底要怎么保存？保存在哪儿？

笔记工具的需求

  • 资料需要管理和维护才能更有意义。
  • 然而时间最终还是有限的，也许很多时候不值得花时间分类整理。
  • 只需要一个大的分类即可，分清原创和收藏即可？
  • 需要有搜索功能，终究找资料的时候还是要用搜索来找。
  • 选择太多，保存得太多，为了免费，不断地切换笔记软件，造成很多不必要的成本。
  • 个人喜好：保存为 markdown 即 *.md 文本格式，并且支持 vim 快捷键功能。
  • 免费空间要足够大。
  • 多端同步，手机至少可以读取访问。

选择心路历程

  • 最早也最流行的选择：EverNote，毕竟自己更程序员思维，所以感觉很多细节并不适合程序员，而比较适合普通大众使用者，如 Markdown 的支持，vi 快捷键，无限空间，本地文档目录等。
  • 后转为用 GitHub 的空间。适合发表文章，但一些临时记录的内容，就不太适合放上去了。
  • 使用有道云笔记，来满足日常的文档记录使用，基本符合需求，但感觉软件有些细节做得不满意，如：
    • 搜索有时会感觉结果不准确。
    • 空间较小，6.5G，带广告。
    • 笔记格式奇怪，不支持本地目录。
    • 有时会出现同步冲突现象，产生多个备份文件，要手工删除，比较烦人。
  • 研究用坚果云/OwnCloud同步文档目录，然后用 markdown 带 vi 快捷键的软件来编程的解决方案，目前备选：
    • VSCode，编程常用，且插件丰富，支持打开任意本地文本文件，支持 vi 快捷键。
    • Typora，界面漂亮，特别是有个 Typewrite 编辑模式，写文章感觉非常好，功能也齐全，支持任意的本地文档目录打开 md 文件，唯一缺陷不支持 vi 快捷键。
    • Boostnote，功能和操作基本满意，问题是文档保存为 cson 格式，而不是 md 格式，无法用其他 markdown 软件直接编辑，另外只能一个一个导入原来的 markdown 文档。
    • VNote，界面难看，是 C++ 和 QT 开发的，目录结构也比较受限制，用自己的_vnote.json文件管理所有文章和目录，因此无法识别本地目录的 md 文件，也没有快捷导入功能。
