校园微信打印解决方案

几个方案和思路

  • 完善之前的打印程序，使其更稳定，更可维护和扩展；
  • 使用易云打印接口，替换目前自己开发的客户端；
  • 用更稳定的技术框架重新开发客户端及服务端；（未来可尝试思路）

测试效果

  • 安装易云打印客户端到打印店的 windows 电脑上；
  • 运行 CloudPrintClient 程序；
  • 使用我们的用户名（手机号）和密码登录，勾选记住密码和自动登录；
  • 添加打印店打印机；
  • 添加界面锁密码（可以统一并告知打印店主），关闭后再重启；
  • 云打印客户端会缩小在右下角，服务会连接到易云服务器；
  • 此时再双击小图标会要求输入界面锁密码，输入后可以添加或删除打印机，不过一般店主就不需要改了；
  • 通过易云小程序登录商家账号，即可远程打印文件了；

接口打印

  • 同理可以通过调用相应的接口进行远程打印；
  • 用我们的用户名（手机号）和密码登录易云管理后台，点击开发者中心；
  • 便可以看到添加的打印机，再点击“申请API key”就可以看到一个字串；
  • 通过接口文档中的 URL 发送一个 Post 请求即可远程打印；
  • 客户的信息和上传的文件均可以保存在我们的服务器上；
  • 仅发送打印相关的信息给易云打印接口即可，如：文件地址、是否彩色、是否旋转、是否缩放、打印尺寸、份数、页边距、回调地址等；
  • 上传的文件可以通过我们的服务端程序控制，临时开放一个下载地址，待打印成功后收到回调请求时关闭下载地址即可。
  • 可能需要在微信公众号或微信小程序端开发一个打印预览和调整的功能，增加使用者的体验。

利弊分析

  • 弊：打印店需下载易云打印的客户端，不利于我们的品牌形象统一，另外，懂行人会看出核心的技术路线；
  • 利：由易云打印第三方提供客户端及打印服务的稳定及服务器的支撑，后期即使打印请求压力增大，我们也只需要支付相应的服务费用即可，不用自己维护相应的服务器等。
  • 弊：打印服务由第三方提供，因此可能是我们不可控的因素之一，如果该公司出问题，会对我们的业务有连带的影响。

零点云打印

  • 官网
  • 开放 API
  • 不适合，更针对于餐饮零售打印小票。

技术选型

目前技术框架

  • 服务端采用 ThinkPHP 框架。
  • 公众号是 ThinkPHP 中的 wechat 目录的页面。
  • 客户端是 C# 调用第三方动态链接库 dll 实现打印机驱动和打印界面等。
  • 管理平台是 ThinkPHP 的 admin 目录的页面。
  • 数据库采用 MySQL ，使用 phpMyAdmin 管理。

未来技术选型

  • 考虑采用微服务架构 Dapr，可容纳各种技术栈开发。
  • 服务端考虑采用 Node.js，使用 TypeScript 语言开发。
  • 网页界面考虑采用 Svelte 技术，或者 uni-app 框架，使用 JavaScript 或 TypeScript 开发。
    • MiniProgramStudio 支付宝小程序开发工具
  • 客户端可以考虑采用 Electron (Electron-Vue) + Neon 或 另一款跨平台 GUI 开发框架。
    • 使用Rust + Electron开发跨平台桌面应用
    • 使用Golang跨平台GUI框架进行桌面应用程序开发
    • MonoDevelop
  • Catalyst，这一框架允许制作了 iPad 应用的开发人员将其快速引入 macOS
  • CI 可以考虑用 flow.ci
