Manjaro 系统使用记录

2019年12月20日
| 软件与技术 实践与经验
| 阅读

前几天（今天是2019年12月20日）把 Deepin 系统从 D 盘（1T 硬盘）上删除了，在固态硬盘上开出了 80G 硬盘空间安装个 Manjaro KDE 玩儿一下，中间因为分区操作失误，差点把整个 D 盘给弄丢了，还好冷静地找办法解决，试了很多工具，其中 DiskGenius 是可以找回分区表的，但到最后一步居然提示要收费，好像还要两三百吧，不甘心，又去找免费工具，终于找到个 TextDisk 居然可以用，只是操作界面对国人不友好，还好认识英文，仔细看说明一步步完成了恢复分区的操作，有惊无险，救回了 D 盘。

装好 Manjaro KDE 之后还是很开心的，因为同时顺便解决了原来 Deepin 安装在 D 盘时的一个启动问题， Windows 启动是在 UEFI 模式，而 Deepin 可能因为是在 D 盘启动模式偏不是 UEFI，而是 Legacy 模式，于是每次想切换系统时，还需要进一次 Bios 进行操作一下，好在小米的 Bios 界面还比较友好，感觉还算能接受吧。 而安装好 Manjaro 后，启动界面自然变成了 Manjaro 的暗绿色，上下键头就可以切换两个系统，心里着实暗爽！

遇到几个问题：

  • 输入法的问题，安装了 fcitx 但是切换时无效，一直是英文输入，找到网上说还要配置 .xprofile 才行，感觉这系统确实不是纯白玩儿家玩儿的，对国人不太友好啊。
  • 再就是这个 vi 也有点不太习惯，看来还是得安装 vim 才舒服，默认都没装的。按 Backspace 键不是删除前一个字符，而是临时保留着前面的字符，以为是 Insert 键的开关问题，试着点过好像也不是，最终效果倒是一样，只是不太习惯。
  • 还有就是这个 vi 的上下左右键也是完全不能用了，只能使用 jklh 来移动了，这真的是强制使用 vi 的操作习惯啊！不熟悉 vi 的完全没法玩儿了。
  • 系统默认安装的是 Firefox ，我还正暗自窃喜，因为前段时间正好把习惯从谷歌浏览器转换到 Firefox 了，毕竟国内使用 Firefox 还是方便些，装插件同步什么的都方便，而且现在 Firefox 底层使用 Rust 重写了感觉上是不是也能更快些？人的喜好就是这样容易受心理作用的，不必深究，喜欢就好，可是没想到同步也用不了，分中国版和国际版，默认装的是国际版，再安装中国版感觉还挺麻烦的……主要是想完美切换中英图标什么的，参考这篇文章就可以找到办法了，需要去修改 /usr/share/applications/firefox.desktop 文件的内容。

备份系统：

# rsync -aAXv --exclude={"/dev/*","/proc/*","/sys/*","/tmp/*","/run/*","/mnt/*","/media/*","/lost+found"} / /path/to/backup/folder

尝试 dwm 窗口管理：

  • Tiling Window Managers, and Getting Started with dwm

体验了一下，感觉上手还是有些门槛的，很多基本的操作不熟悉，短时间还没法上手工作。

添加一个新桌面才能登录使用

需要添加一个 dwm.desktop 的文件，具体可参考Dwm not showing in lightdm log in screen，这样重启后，在登录系统的界面左下角会有下拉菜单选择，选择 dwm 即可进入了，不过基本上需要配置好才能使用，新手完全玩儿不起来，需要学会怎么用，再就是要一点点配置到自己舒服才能顺畅使用。

sudo vi /usr/share/xsessions/dwm.desktop

[Desktop Entry]
Encoding=UTF-8
Name=dwm
Comment=dwm window manager
Exec=/usr/local/bin/dwm
Type=Application

尝试 ranger 功能：

  • 开始用时想编辑某个文件，按 E 结果没反应，退出后发现有报错提示“没找到 vim”，因为想用 nvim ，已经装好了，把 vi 和 vim 都链到 nvim 了，不想再装 vim，找了些资料，如下：
    • Ranger Man 中 ENVIRONMENT 里面提到的 EDITOR 是用来设置默认的编程器的，所以想到修改 .xprofile 应该可以，于是添加了一行 export EDITOR=nvim，重启电脑后果然可以了！
